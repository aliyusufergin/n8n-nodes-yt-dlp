# Separate node and toolchain licensing

The main TypeScript node package will use the MIT license, while each platform package will use GPL-3.0-or-later and preserve the packaged tools' license texts, versions, checksums, and corresponding-source references. Every npm release will have a version-specific Corresponding Source Bundle attached to its GitHub Release containing exact source snapshots, build recipes, patches, configuration, component inventory, and manifest. Platform packages will carry the GPL text, third-party notices, and a `SOURCE_OFFER.md` with an immutable bundle link; CI will block binary publication until these materials exist, and they will remain available while the binaries are distributed. Keeping the executables in separate packages and communicating with them only through subprocess arguments maintains a clear boundary between the permissively licensed integration code and the GPL-covered toolchain. Commercial distribution is not currently planned; any future commercial distributor should obtain its own legal review, while noncommercial public distribution still follows the same source-access obligations.
