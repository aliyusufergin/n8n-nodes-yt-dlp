# Classify every supported option

Each packaged yt-dlp version will have an option catalog covering every accepted option, alias, value form, and node-controlled setting. Ordinary cataloged options will retain their CLI order and spelling, while restricted options and attempts to override the packaged executables, JavaScript runtime, config and plugin isolation, credential paths, or execution workspace will be rejected or replaced by node-owned values. Raw traffic and page debugging through `--dump-pages`, `--write-pages`, `--load-pages`, `--print-traffic`, and their historical aliases will also be restricted because base64 output and unmodified artifacts can bypass secret redaction; ordinary verbose diagnostics remain available. `--enable-file-urls`, case-insensitive file URLs, local-path-like positional inputs, and stdin as a positional source will be rejected so an invocation cannot read container files outside its workspace. Options that introduce a nested argument language will not bypass the catalog: user-defined aliases and raw postprocessor or downloader arguments will be restricted, fixed preset aliases must have recursively classified expansions, and compat-option values will be versioned and allowlisted. Unknown options will fail before process creation. An update cannot pass its release gate until every added or semantically changed upstream option has been reviewed and classified, preventing a toolchain update from silently expanding the node's authority.
