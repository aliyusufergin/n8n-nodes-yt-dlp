# Keep execution inside the packaged toolchain

The node will accept an argument line rather than a complete shell command and will launch only the packaged yt-dlp executable without a shell, stdin, or a terminal. Stdout and stderr will be pipes. It will reject restricted options such as `--exec`, external downloaders, plugin loading, executable-location overrides, and option values that request interactive prompting; authentication combinations that could prompt must instead be complete in the Secrets credential. Sites requiring a captcha, confirmation, or other interaction will fail rather than receive prompt emulation. The node will also ignore host configuration. This preserves yt-dlp's non-interactive download and transformation vocabulary while preventing shell syntax or yt-dlp options from turning the node into a general-purpose process executor.
