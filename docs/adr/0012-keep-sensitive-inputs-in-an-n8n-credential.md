# Keep sensitive inputs in an n8n credential

Cookie contents and sensitive CLI options will be supplied through an optional encrypted yt-dlp Secrets credential rather than the workflow's argument line. The node will materialize cookie content only inside the execution workspace, apply the same argument restrictions to sensitive options, and reject host credential sources such as browser profiles and netrc. Before process creation it will derive a secret set from sensitive option values, parsed cookie values, and permitted proxy credentials. A streaming redactor will replace their plain, URL-encoded, and JSON-escaped forms, structured sensitive header values, URL user information, and recognized sensitive query values before captured output is bounded or exposed. Neither argument line will be echoed, and node logs will contain only neutral invocation diagnostics. Documentation will state that transformed secrets cannot be exhaustively recognized and that requested artifacts—including info JSON—are neither scanned nor modified and may contain private data. Temporary secret material will be removed during cleanup.
