# Separate results from download artifacts

The node will expose a Result output with one execution-result JSON item per yt-dlp invocation and an Artifacts output with one binary item per final download artifact. A Result has a `succeeded`, `failed`, or `timed_out` status; nullable exit code, signal, and redacted error summary; duration; bounded stdout and stderr with byte counts and truncation flags; artifact count; and packaged component versions. It never echoes either argument line and never implicitly parses stdout. Each Artifact item uses the `data` binary property and describes its relative path, filename, extension, media type, and size in JSON. Both outputs preserve `pairedItem` linkage to the source input. This keeps metadata-only invocations useful and makes files composable with downstream n8n nodes without duplicating secrets into execution data.
