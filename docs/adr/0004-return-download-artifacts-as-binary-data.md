# Return download artifacts as binary data

Each invocation will use an isolated workspace with separate output, temporary, and node-private areas. The node will control yt-dlp's root and temporary paths, reject path overrides and stdout media output, and allow output templates only when their resolved paths remain inside the output area. Every regular file remaining there after processing—including requested media, subtitles, thumbnails, descriptions, links, and metadata—will become a separate n8n binary item with its relative path and media type. Temporary fragments, partial downloads, credentials, node-private files, symbolic links, sockets, and other special files will never become download artifacts. Completed files from a failed invocation will be emitted only under Continue On Fail. The node will stream accepted artifacts into n8n's binary-data storage and then remove the workspace, enabling downstream nodes to consume downloads without a mounted volume or accumulated container files.
