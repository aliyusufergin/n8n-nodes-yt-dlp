# Ship the complete media toolchain

The node will ship a packaged toolchain containing yt-dlp, FFmpeg, FFprobe, and required companion assets for each supported architecture instead of discovering host-installed executables. FFmpeg and FFprobe will come unchanged from a pinned release-branch static GPL build rather than master; packaging will omit unrelated executables and development files while retaining licenses, sources, and build references. CI will record packed and installed sizes and verify installation in a clean n8n runtime. Registry or installation-size failure will stop the release rather than trigger a postinstall download or a reduced toolchain. This increases distribution size and carries the bundled projects' license obligations, but makes merging, extraction, conversion, and post-processing work without a custom n8n image or manual system setup.
