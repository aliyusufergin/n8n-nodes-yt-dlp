# Update the toolchain through reviewed package releases

The packaged toolchain will be immutable within an installed package version: yt-dlp self-update options will be restricted, and executable versions will change only in a newly published, tested node and platform package release. Releases will pin a tested yt-dlp nightly build because upstream recommends the nightly channel for regular users and warns that stable builds can become stale as sites change. The master channel will not be packaged. Upstream changes may open an update pull request, but checksum, license, build, and integration checks plus human approval are required before an npm release. This keeps installations reproducible and preserves npm integrity guarantees; the node will report the packaged component versions for diagnosis.
