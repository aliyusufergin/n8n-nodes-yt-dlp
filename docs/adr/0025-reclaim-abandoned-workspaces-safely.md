# Reclaim abandoned workspaces safely

Every invocation workspace will be a randomly named, mode-0700 directory under the system temporary root, with secret files limited to mode 0600 and an owner marker recording the node version, PID, and Linux process-start identity. Normal success, failure, timeout, and cancellation paths will remove it in a finally block after accepted artifacts enter n8n storage. A bounded best-effort scavenger on a later node run may remove only matching prefixed directories whose valid marker proves that the original process is gone or its PID has been reused. Ambiguous ownership leaves the directory untouched and produces a neutral warning; age alone never makes an active workspace removable. An interrupted artifact transfer fails the invocation and triggers cleanup rather than emitting an incomplete binary item.
