# Gate releases with deterministic tests

Required CI will cover argument parsing, the option catalog, path confinement, redaction, bounded output, artifact classification, failure handling, timeout, cancellation, and process-group termination. Local HTTP fixtures will exercise small downloads, playlists, sidecar files, FFmpeg merging, and post-processing without depending on third-party sites. Release jobs will install and run the complete package in the official n8n 2.x Docker image on amd64 and arm64, verify platform selection and tool versions, and run the n8n linter, type checks, unit tests, integration tests, and package-content checks. Any required failure blocks publication. Separate scheduled live-site canaries may create an alert or issue, but will retain no media or credentials and will neither block deterministic CI nor publish a release automatically.
