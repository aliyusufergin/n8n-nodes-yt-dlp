# Release test fixtures and live extractor canary

Accessed: 2026-07-17

## yt-dlp anchor

- Stable documentation/test reference: [`2026.06.09`](https://github.com/yt-dlp/yt-dlp/tree/2026.06.09)
- Release candidates use the exact nightly source/tag frozen by the Toolchain Lock; `2026.06.09` establishes the upstream testing mechanisms, not the final binary pin.

## Findings

1. **Kanıtlanmış platform gerçeği:** yt-dlp's own downloader tests create a loopback HTTP server and exercise ranged, non-ranged, content-length, and no-content-length downloads with generated bytes. Its proxy tests create local HTTP/HTTPS proxy servers, including proxy authentication. Sources: [`test_downloader_http.py`](https://github.com/yt-dlp/yt-dlp/blob/2026.06.09/test/test_downloader_http.py) and [`test_http_proxy.py`](https://github.com/yt-dlp/yt-dlp/blob/2026.06.09/test/test_http_proxy.py), yt-dlp 2026.06.09.

2. **Kanıtlanmış platform gerçeği:** yt-dlp's EJS integration tests contain fixed signature/N-challenge vectors, download specific YouTube player JavaScript, and run the vectors through available providers including Deno. These tests are marked as download tests and therefore combine deterministic expected values with mutable external availability. Source: [`test_jsc/test_ejs_integration.py`](https://github.com/yt-dlp/yt-dlp/blob/2026.06.09/test/test_jsc/test_ejs_integration.py), yt-dlp 2026.06.09.

3. **Kanıtlanmış platform gerçeği:** yt-dlp documents FFmpeg/FFprobe, yt-dlp-ejs, and a supported JavaScript runtime as highly recommended; yt-dlp-ejs plus a runtime is required for full YouTube support. The README uses `BaW_jenozKc` as its recurring test-video example. Source: [`README.md`](https://github.com/yt-dlp/yt-dlp/blob/2026.06.09/README.md#dependencies), yt-dlp 2026.06.09.

4. **Kanıtlanmış platform gerçeği:** Upstream's contribution guide provides `hatch test`/`devscripts.run_tests` and requires extractor test URLs, while explicitly asking contributors not to submit AI/LLM-generated issues or pull requests. Running pinned upstream tests as read-only release evidence does not require or authorize any upstream issue, PR, or comment. Source: [`CONTRIBUTING.md`](https://github.com/yt-dlp/yt-dlp/blob/2026.06.09/CONTRIBUTING.md), yt-dlp 2026.06.09.

5. **Lisans/güvenlik riski:** A release gate based only on public media sites is mutable, rate-limitable, geolocation-sensitive, and may involve content with unclear test/redistribution rights. A gate based only on local direct media cannot prove that the frozen extractor/EJS/Deno combination still works against the live service it targets.

6. **Ürün kararı:** Make a hermetic, blocking queue-mode E2E lane the primary acceptance proof. A disposable test-only origin and authenticated proxy, reachable only inside the test network, serve project-generated synthetic media, manifests, HTML/RSS playlist candidates, subtitles, thumbnails, redirects, cookie/basic-auth challenges, slow responses, truncation, and oversized responses. Fixtures include generation recipes, license dedication, exact SHA-256, and bounded sizes; no user or third-party credential/content is used.

7. **Ürün kararı:** Run the exact published tarballs through Community Packages on every ADR 0025 anchor with external egress denied during hermetic workflow execution. Cover direct download, one FFmpeg transformation/merge, FFprobe use, deterministic multi-artifact/playlist behavior, subtitle/thumbnail outputs, cookie and authenticated proxy transport, progress suppression, timeout/cancellation, output/resource limits, binary round-trip, worker propagation/recreation, and cleanup. Any generic-extractor fixture behavior not proven by a controlled prototype remains **doğrulanmadı** and must not be assumed in the final matrix.

8. **Ürün kararı:** Add a separate blocking-but-retriable live extractor/JSC canary at Release Candidate cut. It runs only in an isolated disposable CI runner, uses no credentials, does not download media, captures bounded/redacted diagnostics, and exercises the frozen yt-dlp/EJS/Deno against an upstream-documented test URL/player. It must prove extraction and actual Deno challenge-provider invocation; a network/rate-limit outage is `inconclusive`, not a pass, and `latest` waits for a clean pass.

9. **Ürün kararı:** In the toolchain build gate, run the frozen yt-dlp source's official EJS challenge-vector tests against the packaged Deno/EJS inputs before packaging. This source-level evidence complements but does not replace the published-binary live canary or the full node E2E.

10. **Ürün kararı:** Never call the live canary a supported-site guarantee. Hermetic tests prove the node contract and queue architecture; the live canary proves only that one frozen extractor/challenge path worked at a recorded time. Store URL/test identity, exact tool versions, runner region, timestamp, outcome class, and redacted bounded evidence in the release record.

11. **E2E ile doğrulanacak varsayım:** The exact yt-dlp nightly may change its generic extractor, EJS messages, test vectors, or standard test URL. At RC cut, pin the exact upstream test commit and prove the selected non-download invocation actually reaches Deno. If it does not, the canary design is not accepted and must be revised rather than silently passed.

12. **Cevapsız soru:** The exact synthetic playlist/subtitle/thumbnail fixture format, live canary URL/player, acceptable retry window, and stable evidence string are **doğrulanmadı**. They require a controlled prototype and legal/content review before implementation acceptance.
